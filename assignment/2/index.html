1. Strapi

What is Strapi? Strapi is an open-source, self-hosted headless CMS that prioritizes developer experience and flexibility. It’s popular for its highly customizable API and support for REST and GraphQL APIs

Strapi has a user-friendly interface that makes it easier to manage. the interface is designed so developers can fine it ease to organize and publish all necesary content.

Pros

- Full control

- Flexible with Rest and GraphQL API support.

- Strong support

- Also, open source community

Cons

- Configuration can be very complex

- In the case of self-hosting, maintenance will be required.

References

[Strapi](https://strapi.io/discovery/headless-cms/strapi)

2. Contentful

What is Contentful? Contentful is a headless CMS designed for teams that want a flexible, easy-to-integrate solution. It allows users to create, manage, standardize and deliver high-speed, reliable digital experiences on any channel.

Pros

- Pre-designed and custom filters for easy content search

- URL localized that allow editors to work in different languages and not just the default.

- SaaS reduces the need for maintenance

Cons

- It may not be the best option in terms of money, especially for business functions.

- Limited backend customization compared to self-hosted solutions.

References

- [Contentful](https://strapi.io/discovery/headless-cms/contentful)

3. Sanity

What is Sanity? Sanity is an API-driven, real-time headless CMS that offers extensive customization options and supports collaborative content editing. It uses JavaScript-based schematics, so it is easier for everyone.

Pros

- Real-time Collaboration, very good thing so work in a team

- JavaScript-based schema makes it good for customzation.

- Good for small projects since it has a free tier

Cons

- it has a very steep learning curve due to very unic models and terminologies.

References

- [Sanity](https://strapi.io/discovery/headless-cms/sanity)

4. Prismic

What is Prismic? Prismic is a headless page builder for Next.js, Nuxt and SvelteKit sites. It is known for its simplicity and quick setup.

Pros

- It is easy to get it started

- Multilingual support

- Price is right and it also has free tiers for small projects.

Cons

- API limitations can impact complex or large-scale applications.

- Limited customization compared to fully open source options.

References

- [prismic](https://strapi.io/discovery/headless-cms/prismic)

5. Ghost

What is Ghost? Originally a blogging platform, Ghost has evolved into a headless CMS optimized for creating content-rich sites, with a strong emphasis on publishing and membership features.

What is good about Ghost? Tools are very powerfull for managing posts or subscription models.

Pros

- Easy configuration and administration

- Automated content workflows

- It is good for blogs

- Good for memebership-driven content

Cons

- Not so flexible. It is limited compared to others headless CMS.

- it focused on publishing, so it is not so suitable for complex aplications.

References

- [Ghost](https://strapi.io/discovery/headless-cms/ghost)

While Liquid is is a good templating language, I found it less flexible and more challenging to work with compared to other templating engines like Nunjucks or plain JavaScript. Because of these issues, I created a small example to understand strapi so I could implemented later in my project that I will be changing my templating language, I am thinking to use JS.

Again I created a basic example as well, and I can tell I liked both and both are very good. Once everything gets stared everything seems very clear and easy to use. please see some of the screenshoots I took to show both examples.
