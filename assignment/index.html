Assignment 2 Documentation

Created on 11/22/2024, 11:51:58 PM

This project integrates Eleventy (11ty) and Strapi to demonstrate modern web development practices. It features a Blog section with paginated posts and a Custom Schema section with filtering functionality. Using Strapi's API for content management and Eleventy for static site generation, the project highlights dynamic rendering, responsive design, and scalable architecture.

Step 1: Configure Strapi for Blog and Custom Data

Begin by setting up the content types in Strapi. For the Blog section, create a Content-Type named Blog Post using the Content-Type Builder in Strapi. Add fields such as Title (Text, required), Content (Rich Text, required), Date (Date, required), Tags (Text, repeatable), Author (Text), and Image (Media). Similarly, for the Custom Schema, create a Content-Type named Custom Data and include fields like Name (Text, required), Description (Rich Text, required), Category (Text), and Image (Media). This setup ensures the schema is structured for dynamic content rendering.

Step 2: Add Content in Strapi

Once the schemas are set up, navigate to the Content Manager in Strapi to populate the fields. Add entries to the Blog Post Content-Type, such as lab writeups, career-related posts, or technology-based topics, including rich text content and images for visual appeal. For the Custom Data Content-Type, add items related to categories like "Technology" or "Projects" with relevant descriptions and images. This step prepares the content for fetching and displaying on the website.

Step 3: Fetch Data from Strapi

To access the data, use the Strapi REST API endpoints. Test the endpoints for Blog Post and Custom Data using tools like Postman or cURL. For instance, the endpoint for fetching blog posts might look like http://localhost:1337/api/posts?populate=*. Verify that the API returns the data correctly, which ensures a smooth integration with Eleventy.

Step 4: Integrate Strapi with Eleventy

Modify the .eleventy.js file in your Eleventy project to fetch data dynamically from Strapi. Use node-fetch to call the API endpoints and add the data to Eleventy's collections. For example, create a collection for blog posts by mapping the API response fields to template variables like title, content, and image. Repeat the process for the Custom Data collection. Once set up, test the build process to ensure the data fetches correctly and populates the templates.

Step 5: Create and Style Templates

Design templates to render the fetched data. For the blog section, create a post.njk template to display individual posts dynamically with fields like title, author, date, and content. Similarly, for the custom schema, create a custom.njk template to render items with fields like name, description, and image. These templates define how the data appears on the site.

Step 6: Implement Pagination and Filtering

Add pagination to the blog section by defining a pagination configuration in .eleventy.js, ensuring blog posts are navigable via "Next" and "Previous" buttons. For the custom schema section, implement filtering by category in the custom.njk template, enabling users to view specific subsets of data dynamically. These features enhance the site's usability and interactivity.
